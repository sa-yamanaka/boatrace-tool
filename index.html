使い方ガイド

運用の流れ

STEP1: スコア表で1レース分のデータを入力

公式サイト(boatrace.jp)の出走表を見ながら、各艇の選手名・級別・勝率・モーター2連率・平均ST・F持ち・チルトを入力。レース情報(風・波・会場)も入力します。自動でスコア・推奨買い目が出ます。

STEP2: 「予想記録に追加」ボタンで記録

最初の2週間は「実際に買った点数=0」のまま記録するのが鉄則。買わずに自分のモデル精度を測ります。

STEP3: レース後、結果記録に1着〜3着を入力

本命的中の✓と回収額を入れると損益が自動計算。本命とは「スコア1位の艇」のこと。

STEP4: 集計シートで成績確認

全体・会場別・スコア帯別・判定別の的中率と損益が見れます。「スコア14以上だと的中率8割」みたいな発見があったら、その帯を狙う運用に切り替え。

STEP5: スコア設定で配点を調整

外しているパターンを見て、効いていない項目の配点を下げ、効いている項目を上げます。これがあなた独自のモデル作り。

データの保存

画面上部の「💾 JSONで保存」でデータをJSONファイルにダウンロード。「📂 JSON読み込み」で復元できます。ブラウザ内にも自動保存されてますが、PCが変わったりキャッシュクリアで消えるので、定期的にJSON保存しましょう。「📊 Excelエクスポート」で.xlsx形式の閲覧用ファイルも作れます。

スマホでも見たい時(GitHub Pages運用)

STEP1: GitHubアカウント作成 → リポジトリ作成
  1. github.com にアクセスして無料アカウント作成
  2. 右上の「+」→「New repository」
  3. 名前は boatrace-tool など(英数字)
  4. Publicを選択(GitHub Pages無料利用に必要)
  5. 「Add a README file」にチェック → Create
STEP2: HTMLファイルをアップロード
  1. このHTMLファイル名を index.html に変更
  2. リポジトリ画面で「Add file」→「Upload files」
  3. index.html をドラッグ&ドロップ → Commit changes
STEP3: GitHub Pagesを有効化
  1. リポジトリの「Settings」→「Pages」
  2. 「Source」で main ブランチを選択 → Save
  3. 数分後、URLが表示される: https://(ユーザー名).github.io/boatrace-tool/
  4. このURLをPC・スマホ両方からアクセス可能
STEP4: データ同期(PC → GitHub → スマホ)
  1. PCで「💾 JSONで保存」→ ファイル名を data.json に変更
  2. GitHubリポジトリで data.json をアップロード(同じ場所に上書き)
  3. スマホでツールを開き、「📡 URL読み込み」をクリック
  4. URL欄に https://(ユーザー名).github.io/boatrace-tool/data.json を入力
  5. 「起動時に自動で読み込む」をONにすれば、次回以降は自動同期

※スマホでは「見る」専用。データ更新はPCで行う運用が想定。

STEP5: スマホのホーム画面に追加(アプリ化)
  • iPhone: Safariで開く → 共有ボタン → ホーム画面に追加
  • Android: Chromeで開く → メニュー → ホーム画面に追加

これでアプリのようにワンタップで起動できます。

基礎知識のおさらい

競艇のコース取り基礎

1号艇が最も有利(全国平均1着率約55%)。追い風はまくり有利=1号艇不利、向かい風は1号艇有利。会場ごとに1コース1着率が違うので、会場メモを参考に。

決まり手の種類

逃げ(1号艇1着)・差し・まくり・まくり差し・抜き・恵まれ。1号艇が弱そうなら次にどの決まり手が出るかを想像する。

チルトとは？

ボートのモーター取り付け角度。プラス角度(0.5〜3度)=直線重視(まくり狙い)、マイナス=旋回重視。2号艇以下でチルトが高い選手がいると、1号艇がまくられる可能性が上がる。

避けたほうがいいレース

雨や濃霧の日、B級ばかりの予選レース、前づけが激しいレース、ドリーム戦・優勝戦(実力拮抗で荒れやすい)。逆に企画レース(1〜2R)は固めで初心者向き。

📡 URLからJSONデータを読み込む

GitHub Pagesやクラウドストレージに置いたJSONファイルのURLを指定して読み込みます。

※GitHub Pagesの場合、リポジトリ直下に data.json を置いて、上記URLでアクセスできるようにしてください。CORS設定不要(同一ドメイン or GitHub Pagesは公開なので可)。
